# Release-managed app Release Management

Release-management workspace for the Release-managed app application.
The files in this directory are designed to be readable by the Shiny Release Manager app and by humans during release work.

## Metadata

| Field | Value |
| --- | --- |
| App name | `Release-managed app` |
| App prefix | `SOLARR` |
| Issue prefix | `PKG-SOLARR` |
| Current production version | `0.0.0` |
| Active release branch | `SOLARR-0.0.1` |
| Working branch | `SOLARR-0.0.1` |
| Target release version | `0.0.1` |

## Linked Files

| File | Path | Purpose |
| --- | --- | --- |
| `README.md` | `docs/release-management/README.md` | Project release-management overview and linked file map. |
| `RULES.md` | `docs/release-management/RULES.md` | Operational workflow for versions, branches, commits, pushes, merges, and custom project rules. |
| `TRACKER.md` | `docs/release-management/TRACKER.md` | Open issue register with metadata and field descriptions. |
| `VERSIONING.md` | `docs/release-management/VERSIONING.md` | Closed issue and release register with metadata and field descriptions. |

## Workflow Summary

Use `TRACKER.md` for open issues, `VERSIONING.md` for closed verified work, and `RULES.md` for operational rules.
